# 子供向け足し算・引き算練習Web 仕様書

本書は [01_requirements.md](./01_requirements.md) の要件を実装可能なレベルまで具体化した仕様書である。

## 1. 概要

- 小学一年生向けの足し算・引き算練習Webアプリ。
- 問題を1問ずつ表示し、全20問を解き終えると正解数を表示する。
- スマホ・タブレットでの利用を前提とし、回答はワンタップの数字ボタンで入力する。
- 正解時などに子供の興味をひく楽しいエフェクトを表示する。

### 1.1 モード一覧

| モード | 内容 |
|---|---|
| 単純足し算モード | 一桁の足し算 20問 |
| 単純引き算モード | 一桁の引き算 20問(答えは常に0以上) |
| 文章題モード | 一桁の足し算・引き算になる文章題 20問 |

## 2. 画面仕様

画面はSPA内の3画面で構成する。ページ遷移(URL変更)は行わず、アプリ内の状態で切り替える。

### 2.1 ホーム画面

- アプリタイトルを表示する。
- モード選択ボタンを3つ縦に並べる:「たしざん」「ひきざん」「ぶんしょうだい」。
- ボタンは子供が押しやすいよう大きく(高さ64px以上)、色分けする。
- 文言はすべてひらがなとする。

### 2.2 問題画面

- 上部に進捗を表示する(例: 「5 / 20」およびプログレスバー)。
- 中央に問題を表示する:
  - 計算モード: 数式を大きく表示(例: 「3 + 2 = ?」)。
  - 文章題モード: 文章を表示(例: 「いちごが 3 こ あります。…」)。文字サイズは大きめ、分かち書きで表示する。
- 下部に回答用の数字ボタンをグリッド表示する(仕様は「4. 入力UI仕様」参照)。
- 回答すると正誤フィードバック(「5. エフェクト仕様」参照)を表示し、自動で次の問題へ進む。
- 左上に「やめる」ボタンを置き、タップでホーム画面に戻る(このセッションの結果は保存しない)。

### 2.3 結果画面

- 正解数を大きく表示する(例: 「20もんちゅう 17もん せいかい!」)。
- 正解数に応じたお祝いエフェクトを表示する(「5. エフェクト仕様」参照)。
- ボタンを2つ表示する:
  - 「もういちど」: 同じモードで新しい20問を開始する。
  - 「ホームへ」: ホーム画面に戻る。

## 3. 出題仕様

### 3.1 単純足し算モード

- `a + b` の形式。`a`, `b` はそれぞれ 1〜9 の乱数。
- 答えの範囲は 2〜18。

### 3.2 単純引き算モード

- `a - b` の形式。答えが常に 0 以上になるよう `a ≧ b` で生成する。
- `a` は 1〜9、`b` は 0〜a の乱数(答えの範囲は 0〜9)。

### 3.3 文章題モード

実行時に生成AIは呼び出さない(インフラコスト最小化のため)。開発時に生成AIで作成したテンプレートプールをアプリに同梱し、実行時にテンプレートへ乱数で数値・題材を差し込んでバリエーションを出すハイブリッド方式とする。

- テンプレートは合計100種類(足し算系50種・引き算系50種)を開発時に生成AIで作成し、静的データ(TypeScriptモジュール)としてアプリに同梱する。
- テンプレートはプレースホルダを含む文字列とする。
  - 例(足し算): 「{item}が {a} {counter} あります。そこへ、あたらしく {item}が {b} {counter} ふえました。{item}は ぜんぶで なん{counter} に なりましたか?」
  - 例(引き算): 「{item}が {a} {counter} あります。そのうち {b} {counter} たべました。{item}は なん{counter} のこって いますか?」
- 題材(`{item}`)は小学生が興味を持つものを用意する(例: いちご、みかん、あめ、どんぐり、シール、ふうせん、きんぎょ、くるま)。題材ごとに助数詞(`{counter}`: こ、まい、だい など)を対応付ける。テンプレートの動詞(たべる、あげる等)と整合する題材のみを組み合わせる。
- 数値 `a`, `b` は 3.1・3.2 と同じルールで生成する。
- 1セッション20問は足し算系・引き算系を混合して出題する(各10問、順序はランダム)。
- 文はひらがな中心・分かち書きとし、小学一年生が読めるようにする。

### 3.4 共通ルール

- 1セッション内で同一の問題(計算モードでは同一の式、文章題では同一のテンプレート×数値の組)は出題しない。
- 20問を生成してからセッションを開始する。

## 4. 入力UI仕様

- 回答は数字ボタンのワンタップで確定する。確定ボタンやテキスト入力は設けない。
- ボタンの数字範囲は、その問題の答えの取りうる範囲に合わせる:
  - 足し算(文章題の足し算含む): 0〜18 を表示(グリッド表示)。
  - 引き算(文章題の引き算含む): 0〜9 を表示。
- ボタンは指で押しやすいサイズ(最小 48×48px、間隔 8px 以上)とする。
- タップ後は正誤フィードバック表示中の誤操作を防ぐため、全ボタンを一時的に無効化する。

## 5. エフェクト仕様

### 5.1 正解時

- 大きなマル(⭕)を表示し、紙吹雪などの楽しいアニメーションを再生する(約1秒)。
- 「せいかい!」「やったね!」などの短いメッセージをランダムに表示する。
- 効果音を鳴らす(端末設定でミュート可能。初期実装では任意)。

### 5.2 不正解時

- バツ(❌)と正しい答えを表示する(例: 「こたえは 5 だよ」、約1.5秒)。
- 子供を責めない・怖がらせない演出とする(派手な失敗音や赤の点滅は使わない)。

### 5.3 結果画面

- 正解数に応じて演出を変える:
  - 20問正解: 特別なお祝い演出(大量の紙吹雪+トロフィー表示など)。
  - 15問以上: 紙吹雪+星の演出。
  - それ未満: 「よくがんばったね!」の応援演出。

## 6. データ保存仕様(LocalStorage)

DBは使用せず、ブラウザの LocalStorage にプレイ履歴を保存する。

- キー: `math-practice/history`
- 値: プレイ履歴の配列(JSON文字列)。1セッション終了ごとに1件追記する。

```json
[
  {
    "playedAt": "2026-07-23T10:30:00+09:00",
    "mode": "addition",
    "correctCount": 17,
    "totalCount": 20
  }
]
```

- `mode` は `addition` | `subtraction` | `word-problem`。
- 履歴は最新100件まで保持し、超過分は古いものから削除する。
- 途中で「やめる」を押したセッションは保存しない。
- 履歴の集計・グラフ表示は将来拡張とする(本仕様の範囲外)。

## 7. 技術仕様

- **フロントエンド**: Vite + React(TypeScript)による静的SPA。
- **ホスティング**: Cloudflare Pages に静的デプロイ。Workers・DB・サーバーサイド処理は使用しない(インフラコスト最小化)。
- **データ保存**: LocalStorage のみ(「6. データ保存仕様」参照)。
- **対応端末**: スマホ縦持ちを基準にデザインし、タブレットにも対応するレスポンシブレイアウトとする。
- **対応ブラウザ**: iOS Safari / Android Chrome の最新版を主対象とする。
- **オフライン動作**: 静的アセットのみで動作するため、初回ロード後は通信不要。
